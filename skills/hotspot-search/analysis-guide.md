# 热点分析框架指南

## 分析维度

### 1. 相关性评分 (Relevance Score)

**评分范围：0-100**

**评分标准：**
- 90-100：高度相关，关键词精确匹配
- 70-89：中度相关，关键词部分匹配
- 50-69：弱相关，关键词间接相关
- 0-49：不相关，过滤掉

**评分因素：**
- 关键词出现频率
- 关键词出现位置（标题/正文）
- 语义关联性
- 上下文匹配度

### 2. 重要性级别 (Importance Level)

**级别定义：**
- `urgent`：紧急，突发重大事件
- `high`：高，重要热点事件
- `medium`：中，一般热点
- `low`：低，普通资讯

**判断因素：**
- 内容时效性
- 影响范围
- 社会关注度
- 媒体报道程度

### 3. 热度分数 (Heat Score)

**评分范围：0-100**

**计算方式：**
```python
heat_score = (
    view_count_weight * normalized_views +
    comment_count_weight * normalized_comments +
    like_count_weight * normalized_likes
)
```

**权重设置：**
- 浏览量：40%
- 评论数：35%
- 点赞数：25%

## 筛选策略

### 时效性筛选

- 默认只显示 90 天内的内容
- 可配置时间范围
- 微博热搜：只显示当前热点

### 质量筛选

- B站：播放量 ≥ 300
- 跳过广告内容
- 跳过非热点内容（百科、教程等）

### 去重策略

- 基于 URL 去重
- 基于标题相似度去重
- 保留最高质量版本

## 排序策略

**主排序：相关性分数降序**

**次级排序：**
1. 热度分数
2. 发布时间（新近优先）
3. 重要性级别

## 评估指标

### 搜索质量

- 准确率：相关结果占比
- 召回率：覆盖热点的程度
- 时效性：结果的新鲜度

### 性能指标

- 响应时间
- 成功率
- 错误率

## 优化建议

### 持续优化方向

1. **数据源扩展**
   - 添加更多信息源
   - 支持 RSS 订阅
   - 支持社交媒体 API

2. **分析精度**
   - 集成 AI 模型
   - 语义分析优化
   - 情感分析

3. **个性化**
   - 用户偏好学习
   - 自定义权重
   - 推荐算法
