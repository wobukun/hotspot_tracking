# AI 热点监控工具 - 需求文档

## 1 引言

### 1.1 编写目的

本文档旨在详细描述 AI 热点监控工具的功能需求、非功能需求及开发规划，为项目的设计、开发、测试、验收和交付提供统一的依据。文档明确了系统的核心功能、技术要求和业务逻辑，确保开发团队能够准确理解并实现预期功能。

### 1.2 项目背景

在信息爆炸的时代，快速获取和分析热点信息对于舆情监控、市场研究、内容创作等领域至关重要。传统的信息获取方式存在效率低、信息碎片化、真假难辨等问题。AI 热点监控工具通过自动化采集、AI 分析和实时推送，解决了这些痛点，为用户提供及时、准确、全面的热点信息。

### 1.3 适用范围

#### 包含范围

- 关键词管理模块：添加/删除监控关键词、激活/暂停单个关键词、AI 自动扩展查询变体、B站账号智能检测
- 热点采集与分析模块：8+ 数据源并行采集、AI 内容真实性验证、AI 相关性评分、AI 重要性分级、AI 智能摘要生成、AI 相关性分析理由
- 信息展示与筛选模块：热点雷达仪表盘、5 种排序方式、多维度筛选、分页加载、展开/折叠分析详情、一键跳转原文
- 通知系统模块：WebSocket 实时推送、邮件通知、站内通知管理
- 全网搜索模块：全网关键词搜索、多数据源聚合搜索结果
- Agent Skills 模块：完全自包含的 AI 技能包、支持多工具

#### 排除范围

- 用户登录和权限管理（本次不开发）
- 历史数据迁移（延后迭代）
- 第三方系统集成（独立开发）

## 2. 功能需求

### 2.1 热点采集与分析

#### 2.1.1 支持的信息源（8+ 个）

| 序号 | 信息源            | 抓取方式                      |
| -- | -------------- | ------------------------- |
| 1  | Twitter        | TwitterAPl.io，Twitter高级搜索 |
| 2  | Bing           | Axios + Cheerio，网页爬虫      |
| 3  | Google         | Axios + Cheerio，网页爬虫      |
| 4  | DuckDuckGo     | Axios + Cheerio，网页爬虫      |
| 5  | Hacker News    | HackerNews Algolia API    |
| 6  | 搜狗搜索           | Axios + Cheerio，网页爬虫      |
| 7  | B 站 (Bilibili) | B站公开API，视频搜索和账号检测         |
| 8  | 微博             | 第三方 API                   |

#### 2.1.2 采集功能

- 8+ 数据源并行采集
- 系统每 30 分钟自动从各信息源抓取内容
- 支持启用/禁用特定信息源
- 支持设置每个信息源的抓取频率
- 支持配置代理（针对需要代理的信息源）
- Python 采集脚本

### 2.2 关键词管理

#### 2.2.1 关键词配置

- 添加/删除监控关键词
- 激活/暂停单个关键词
- AI 自动扩展查询变体
- B 站账号智能检测
- 支持关键词分组管理
- 支持同义词配置（如"AI"和"人工智能"关联）
- 支持排除词配置（如排除"广告"相关内容）

#### 2.2.2 关键词匹配规则

- 精确匹配
- 模糊匹配
- 正则表达式匹配（高级模式）

### 2.3 AI 分析引擎

#### 2.3.1 真假识别

- 基于 OpenRouter API 调用 AI 模型 (DeepSeek、Claude、GPT 等)
- 对新闻/内容进行真实性评估
- 标记可疑或低可信度内容
- 评分机制（0-100可信度分数）

#### 2.3.2 相关性分析

- 判断内容与用户关键词的相关程度
- 过滤无关内容
- 相关性评分（0-100相关性分数）
- AI 相关性分析理由

#### 2.3.3 内容摘要

- AI 智能摘要生成
- 提取关键信息和观点
- 识别内容情感（正面/负面/中性）

#### 2.3.4 重要性分级

- AI 重要性分级
- 基于多维度评估内容重要性
- 重要性级别：urgent / high / medium / low

#### 2.3.5 AI 模型配置

- 支持切换不同 AI 模型
- 支持自定义分析提示词
- 支持批量分析配置

#### 2.3.6 Query Expansion

- AI 驱动的查询扩展
- 自动生成关键词变体
- 提高搜索覆盖范围

### 2.4 实时推送系统

#### 2.4.1 WebSocket 推送

- 实时推送新抓取的内容
- 推送 AI 分析结果
- 推送系统状态和通知
- 支持断线重连

#### 2.4.2 推送内容

- 原始内容链接
- 内容摘要
- AI 可信度评分
- AI 相关性评分
- 情感分析结果
- 发布时间和来源

#### 2.4.3 推送控制

- 支持设置推送频率阈值
- 支持关键词级别推送开关
- 支持免打扰时间段设置

### 2.5 通知系统

#### 2.5.1 WebSocket 实时推送

- 实时推送新抓取的内容
- 推送 AI 分析结果
- 推送系统状态和通知
- 支持断线重连

#### 2.5.2 邮件通知

- 邮件通知 (high/urgent)
- 达到一定数量的新热点时
- 发现高可信度（>90%）的热点时
- 定时汇总报告（每日/每周）
- 系统异常告警

#### 2.5.3 站内通知管理

- 站内通知管理
- 已读/未读状态
- 通知历史记录

#### 2.5.4 邮件内容

- 热点摘要列表
- 重点内容高亮
- 包含内容链接和来源
- 可视化数据图表

#### 2.5.5 邮件配置

- 支持 SMTP 配置
- 支持 TLS/SSL
- 支持多个收件人
- 支持邮件模板定制

### 2.6 信息展示与筛选

#### 2.6.1 展示功能

- 热点雷达仪表盘（实时统计数据）
- 5 种排序方式（最新发现/最新发布/重要程度/相关性/热度综合）

- 分页加载
- 展开/折叠相关性分析详情
- 一键跳转原文

### 2.7 全网搜索

#### 2.7.1 搜索功能

- 关键词全网搜索
- 多数据源聚合结果
- 搜索结果筛选排序

### 2.8 Agent Skills

#### 2.8.1 功能

- 将热点监控能力封装为标准的 Agent Skills
- 安装后在 Cursor、VSCode Copilot、Claude Code 等 AI 编程工具中都能使用

### 2.9 数据存储与管理

#### 2.9.1 存储内容

- 关键词配置（分组）
- 抓取的原始内容
- AI 分析结果
- 系统日志

#### 2.9.2 数据管理

- 支持数据导出（JSON/CSV）

## 4. 非功能需求

### 4.1 性能需求

- 页面加载时间 < 3 秒
- WebSocket 推送延迟 < 1 秒
- 支持 100+ 并发连接
- 批量抓取时不影响用户体验

### 4.2 安全需求

- 敏感配置加密存储
- 防止 SQL 注入和 XSS 攻击
- 请求频率限制

### 4.3 可靠性需求

- 服务 99.9% 可用性
- 自动错误重试机制
- 数据定期备份
- 支持服务重启后恢复

### 4.4 扩展性需求

- 模块化设计，易于功能扩展
- 支持新增信息源
- 支持自定义 AI 模型
- 支持插件系统（未来）

### 5.3 优化前端页面

- Git 版本提交（防止改动破坏已有功能）
- 使用 Agent Skills（UI UX Pro Max）美化界面
- 人工指定 UI 风格（科技感提示词）
- 集成 Aceternity UI 组件库（流星、光影、聚光灯等）

### 5.4 优化信息获取来源

- 分析现有问题（Twitter 回复、信息源单一）

- 增加国内信息源（搜狗、B站、微博）
- B站账号检测和视频获取
- 新鲜度过滤（7天以内）
- 多轮迭代优化信息质量和丰富度的平衡
- 手动提供正确的 API 文档给 AI

### 5.5 信息流筛选和排序

- 让 AI 设计筛选和排序维度
- 多维度筛选（来源/重要性/关键词/时间范围）
- 多维度排序（热度综合/相关性/时间/重要程度）
- 主页和搜索页面同步支持筛选排序
- 分页功能
- 排序正确性测试（让 AI 编写单元测试）
- Bug 修复（界面遮挡等问题）

### 5.6 优化热点信息展示

- 增加帖子发布时间和抓取时间
- 增加互动数据（点赞、转发、回复、浏览量等）
- AI 相关性分析理由展示
- 展开/折叠分析详情
- 一键展开/折叠所有帖子
- 更多展示信息优化（作者信息、认证状态）

### 5.7 优化 AI 分析准确度＋扩展思路

- 分析 AI 审核问题（相关性偏差、摘要不精准）
- 实现 Query Expansion 查询扩展

- 建立测试和评估机制（AI 编写单元测试）

- 扩展思路分享
